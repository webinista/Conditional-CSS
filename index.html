What is Conditional CSS?

What is Conditional CSS?

CSS features and their associated JavaScript APIs that let us apply styles or behaviors when a particular condition is met.

A new way of thinking about some of the things we already know and use. Defined by the CSS Conditional Rules Module Level 3. Using broader definition here, to include @viewport which is a separate spec. Some of this is old news. Some of this is very much experimental.

Conditional CSS: a few @rules

  • @media
  • @supports
  • @viewport
  • @document

Includes our old friend @media which is part of the CSS Level 2 specification, and newer features and APIs:

  • Media queries which extends @media.
  • @supports offers native, CSS-based feature detection
  • @viewport applies styles based on the document's viewport
  • @document applies styles based on the document's URL or URL pattern

These also have DOM and CSSOM-based APIs. All of it can be manipulated with scripting.

@media

@media: a recap

Set styles for a particular media type.

@media print {
    body { font-size: 10pt }
}
@media screen, print {
    body { line-height: 1.2 }
}

Media queries

@media and media queries

Applies styles for particular media features.

  • width
  • height
  • device-width
  • device-height
  • orientation
  • aspect-ratio
  • device-aspect-ratio
  • color
  • color-index
  • monochrome
  • resolution
  • scan
  • grid

Media queries basic syntax

@media [only | not] <media_type> [and] (expression){
    selector{
        property: value;
    }	
}
* Only or not for this particular media type, or/and an expression. * Expressions consist of the feature you wish to query, and usually a minimum, maximum, or exact value. * Within the media rule are your selectors and property-value pairs.

Media queries syntax example

@media screen and (max-width: 400px){
        header{
                position:fixed;          
        }	
}

@supports

Applies CSS only when the property is supported and the value is valid and supported.

	/* CSS rules to apply */
}

@viewport

<meta name="viewport">

  • First introduced in iOS Safari
  • Allowed developers to constrain the dimensions of a page to a particular size
  • Divorces the document viewport from the physical viewport

@viewport

  • Introduced by Opera as CSS Device Adaptation
  • Moves viewport information from the HTML document to CSS.

Was originally part of the CSS Conditional Rules Module Level 3 (http://www.w3.org/TR/css3-conditional/), but was pushed to Level 4.

@document

  • Apply styles based on the document URI: per-domain or per-page.
  • Created with a user style sheets use case in mind.
Issues to be worked-out for Level 4 involve other use cases.
  • Might be useful for browser vendors to ship site patches for users.
  • Possibility of using it to organize styles by domain.
