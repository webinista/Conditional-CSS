What is Conditional CSS?

A new way of thinking about some of the things we already know and use. Parts are also defined by the CSS Conditional Rules Module Level 3.

What is Conditional CSS?

CSS features and their associated JavaScript APIs that let us apply styles or behaviors when a particular condition is met.

Conditional CSS: the @rules

  • @page
  • @media
  • @supports
  • @viewport
  • @document

Includes our old friends, @media and @page, which were part of the CSS Level 2 specification, and newer features and APIs:

  • Media queries, Generated Content for Paged Media extend @media.
  • @supports offers native, CSS-based feature detection
  • @viewport applies styles based on the document's viewport
  • @document applies styles based on the document's URL or URL pattern

Some of these also have JavaScript or DOM APIs, so you can use them with scripting.

@media: a recap

Set styles for a particular media type.

@media print{
    body { font-size: 10pt }
}
@media screen, print {
    body { line-height: 1.2 }
}

Media queries

@media and media queries

Applies styles for particular media features.

  • width
  • height
  • device-width
  • device-height
  • orientation
  • aspect-ratio
  • device-aspect-ratio
  • color
  • color-index
  • monochrome
  • resolution
  • scan
  • grid

Worth mentioning that you should use content-based breakpoints for your site rather than device-width based breakpoints.

Don't set breakpoints based onparticular device widths where you use known widths for for "desktop," iPad, Nexus 7, or the iPhone, etc. Set them at the point where your content no longer looks good.

Easier to do with new viewport units (http://www.w3.org/TR/css3-values/), but they're not widely supported by mobile browsers yet. (Source: http://caniuse.com/viewport-units)

Was originally part of the CSS Conditional Rules Module Level 3 (http://www.w3.org/TR/css3-conditional/), but was pushed to Level 4.

@document

  • Apply styles based on the document URI: per-domain or per-page.
  • Created with a user style sheets use case in mind.
Issues to be worked-out for Level 4 involve other use cases.
  • Might be useful for browser vendors to ship site patches for users.
  • Possibility of using it to organize styles by domain.
